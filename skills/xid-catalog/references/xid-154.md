# Xid 154 — Recovery Action Changed

Xid 154 itself is informational. Severity is determined by the `recovery_action` field on the same event in `triage/_data/xid_events.json` (or the raw line in `nvidia/xid_errors.txt` / `logs/journal_kernel.ndjson`).

## Field → severity mapping

```
recovery_action          | severity | action            | notes
-------------------------|----------|-------------------|----------------------------------
none                     | info     | none              | no action
drain p2p                | warning  | drain-p2p         | pause NVLink P2P; resume on clear
drain and reset          | critical | reset-gpu         | drain workloads then reset
gpu reset required       | critical | reset-gpu         | reset immediately
node reboot required     | critical | reboot-node       | full host reboot
<missing or null>        | warning  | escalate-for-review | treat as ambiguous
```

## Parsing rules

1. Look for `recovery_action=` or `"recovery_action":` near the Xid 154 line, same record.
2. Strip surrounding quotes; lowercase; whitespace-normalise.
3. If the value is not one of the canonical strings above, set severity `warning` and `action_unknown: true` in the finding.

## Why this exists

NVIDIA introduced 154 to convey runtime policy changes without inventing a new code per recovery class. The `recovery_action` field is the actual signal; the code is just the envelope.
